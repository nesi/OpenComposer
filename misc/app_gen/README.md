# App Generation Workflow

This scaffold regenerates OpenComposer app files from upstream NeSI metadata in a repeatable way, using Claude to generate SLURM-aware form logic and icons.

## What it generates

For each target app in `apps/<APP>`:

- `manifest.yml` — name, category, description, homepage
- `form.yml.erb` — SLURM job submission form with Claude-generated fields and ERB logic
- `icon.png` — app icon generated by Claude (skipped if one already exists; use `--force-icon` to regenerate)

## How it works

1. Fetch NeSI modules list JSON and support-docs index from upstream.
2. Match the app name against the support-docs index to find its markdown page.
3. If the support doc is absent or contains no code examples, fall back to a web search (requires `claude.web_search_command` in config).
4. Pass the support doc to Claude with a structured prompt to generate form fields, ERB parallelisation logic, and an icon.
5. Write files, skipping any that were hand-edited (missing the `# AI generated` marker on line 1) unless `--force` is used.

## Why it is safe

- Default mode is **preview only**. Nothing is written unless `--write` is passed.
- Hand-edited files (those missing the `# AI generated` marker) are never overwritten unless `--force` is used.
- Icons are preserved unless `--force-icon` is passed.

## Data sources

Configured in `misc/app_gen/config.yml`:

- `sources.modules_list_url` — NeSI modules list JSON (metadata: description, homepage, domains)
- `sources.support_docs_index_url` — GitHub API index of support-doc markdown files
- `sources.support_docs_raw_base_url` — raw base URL for fetching individual support docs

## Prompt templates

The Claude prompts are stored as plain-text templates in `misc/app_gen/`:

| File | Purpose | Placeholders |
| --- | --- | --- |
| `prompt_form.txt` | Generates `form_fields` and `erb_logic` | `<<<APP_NAME>>>`, `<<<SUPPORT_MD>>>` |
| `prompt_icon.txt` | Generates `icon_png_base64` | `<<<APP_NAME>>>`, `<<<DIM>>>`, `<<<FMT>>>`, `<<<DESCRIPTION>>>`, `<<<DOMAINS>>>` |
| `prompt_web_search.txt` | Web search fallback for thin/missing docs | `<<<APP_NAME>>>`, `<<<HOMEPAGE_SECTION>>>` |

Edit these files to tune output quality without touching Python source.

## Commands

Preview one app (no files written):

```bash
python3 misc/app_gen/generate_apps.py --app GROMACS
```

Show generated content inline:

```bash
python3 misc/app_gen/generate_apps.py --app GROMACS --show
```

Write files:

```bash
python3 misc/app_gen/generate_apps.py --app GROMACS --write
```

Force overwrite even for hand-edited files:

```bash
python3 misc/app_gen/generate_apps.py --app GROMACS --write --force
```

Regenerate icon even if one already exists:

```bash
python3 misc/app_gen/generate_apps.py --app GROMACS --write --force-icon
```

Multiple apps:

```bash
python3 misc/app_gen/generate_apps.py --app GROMACS --app AlphaFold --write
```

All apps:

```bash
python3 misc/app_gen/generate_apps.py --all --write
```

## Claude command

Configured in `misc/app_gen/config.yml` under `claude.command`. The command reads the prompt on stdin and must return JSON. A second command `claude.web_search_command` is used for web search fallback and should have web tools enabled.

Expected JSON keys:

- Form generation: `profile`, `form_fields`, `erb_logic`
- Icon generation: `icon_png_base64` (base64-encoded PNG)
- Web search: `docs`, `source`

## Notes for iteration

- Always preview before writing. Use `--show` to inspect generated content inline.
- Use `--force` to overwrite files that have been hand-edited.
- To protect a specific app from any overwrite, remove the `# AI generated` marker from its `form.yml.erb` — the generator will then always skip it unless `--force` is passed.
- Tune prompt quality by editing the `prompt_*.txt` templates. Re-run with `--show` to verify output before committing.
